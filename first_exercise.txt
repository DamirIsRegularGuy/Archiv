Процесс установки БД на СУБД ORACLE:

1) После установки СУБД нужно ввести команду 
     
      dbca  ->  Данная команда активирует установку инстанции БД

2) Далее открывается окно по установке БД, там будут разделы, ниже рассмотрим каждый раздел по отдельности

3) Первым идет "Database Operation" в котором будут различные операции с БД, для создания выбираем "Create a database", другие, например, как "configure an existing database" нужен для изменения параметров уже существующей БД, "Delete a Database", удаляет существующую БД, "Manage Templates" нужно для создания, изменения или удаления шаблонов.

4) Дальше раздел "Creation mode" в котором выбираем "Advanced congiguration", что это значит, это позволяет режим позволяет детально настраивать каждый аспект базы данных, включая параметры памяти, хранилища, сетевых настроек, параметров инициализации и других важных параметров.

5) "Deployment Type" ->  это опция, которая позволяет выбрать тип развертывания базы данных в зависимости от ее назначения и требований. Выбираем вид развертывания - это "General Purpose or Transaction Processing". 

6) "Database Identification" -> это этап, на котором указываются основные идентификаторы и параметры для новой базы данных. Эти параметры необходимы для правильной идентификации и управления базой данных в вашем окружении. 
К ним относятся:
                "Global Database Name" -> Это полное имя базы данных, которое используется для уникальной идентификации базы данных в сети.
                "SID" -> Это уникальный идентификатор для базы данных, который используется для идентификации базы данных на уровне сервера.

7) "Storage Option" -> а этом разделе идет момент с тем как будут хранится файлы БД 

8) "Fast Recovery Option" -> быстрое восстановление БД в случае сбоя. Этот параметр особенно важен для обеспечения высокой доступности и минимизации времени простоя базы данных. Здесь ничего не трогаем.

9) "Network configuration" -> Здесь идет настройка параметров сетевого взаимодействия, которые позволяют клиентам подключаться к базе данных.

10) "Data Vault Option" -> позволяет включить или отключить функцию Oracle Data Vault. Эта функция предоставляет дополнительные возможности для управления безопасностью данных, позволяя контролировать доступ к данным и конфигурации базы данных на более детальном уровне. Опять же, здесь ничего не трогаем.

11) "Configuration Options" -> Здесь можно настроить различные параметры конфигурации, которые определяют производительность, безопасность, управление и другие аспекты работы БД. Тут можно ничего не трогать почти, кроме последнего раздела "Sample Schemas", где идет установка примеров схем для БД.

12) "Management Options" -> Насторойка параметров управления и мониторинга базы данных. Так же ничего не трогаем.

13) "User Credentials" -> Здесь устанавливаем пароль для БД.

14) "Creation Option" -> можно выбрать, как будет создана БД. Этот этап позволяет определить, будет ли БД создана сразу, или будет создан только сценарий для последующего выполнения. Ставим, конечно, create database, для немедленного создания БД.

15) "Summary" -> Здесь можно просмотреть все выше сделанные нстройки.

16) "Progress Page" -> Здесь прогресс установки, и если будут, то показ возможных ошибок

17) "Finish" -> успешная установка БД

После этого необходимо ввойти в БД:
sqlplus / as sysdba

Создать какого-нибудь пользователя:
create user USERNAME identified by PASSWORD;

Выдать пользователю права администратора:
grant dba to USERNAME;

Выйти из sqlplus и войти под новым пользователем:
exit;
sqlplus USERNAME/PASSWORD
