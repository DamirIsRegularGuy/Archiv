Как я понял и как знал - ветки в Git нужды для более удобной работы в проекте,
можно улучшать или что-то добавлять в проект, не боясь затронуть основной, 
работающий код. Так же это полезно, если это какой-нибудь командный проект.
Каждый работает над своей задачей, в определенной ветке и потом они все 
сливаются в итоговый проект.
Git branch - нужен для просмотра имеющихся веток и создание их.
Git push - для внесений изменений, которые произошли с проетом, в репозитории
Git pull - как раз нужен для слияния

mkdir gitttest -> создал директорию для файла
touch nodegit.txt -> сам файл
git init gittetst -> сделал директорию репозиторием
git status -> посмотрел что за файлы есть и какие отслеживаются
git add nodegit.txt -> добавил данные файл в разряд отслеживающихся
git commit -m "first" -> зафиксировал изменения
git branch test -> создал новую ветку(p.s. я ее так и не использовал,
потому что запушил основную ветку master)
git remote set-url origin git@github.com:DamirIsRegularGuy/Archiv.git -> 
определяю репозиторий на ГитХабе, куда надо загрузить файл
git push origin master -> запушил изхменеия в ветке master
